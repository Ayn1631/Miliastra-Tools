# Streamlit UI

- 快速开始

```powershell
# python=3.11
pip install streamlit==1.45.1
streamlit run .\scripts\streamlit_app.py
```

## 图片生成 GIA

“图片生成 GIA”页面支持：

- 上传 `png/jpg/jpeg/webp/bmp`。
- 可视化预览图片。
- 拖拽调整图片解析缩放比例，也可以用加减步进输入精确调整。
- 支持等比缩放和不等比缩放；不等比时可分别调整 X/Y 解析缩放。
- 调整期望总宽度和总高度，使用加减步进输入组件，单位为米，范围 `0.01` 到 `50`。
- 生成尺寸支持两种模式：
  - `调高宽自动计算单个大小`：输入总宽/总高，自动计算每个像素块的 `scale.x/scale.z`。
  - `调单个大小自动计算高宽`：输入每个像素块的宽/高，自动反推最终总宽/总高。
- 可选择锁定图片宽高比。
- 设置最大解析像素数，避免一次生成过多实体。
- 当目标尺寸过小导致单像素尺寸低于 `0.01m` 时，会自动继续降采样，保证生成单位缩放仍在 `0.01` 到 `50` 范围内。
- 设置 alpha 透明度过滤阈值。
- 可指定背景 RGB 和容差，从图片四周出发用 BFS 扣掉连通背景，不越过主体外围轮廓。
- 生成算法支持：
  - `逐像素生成`：每个可见像素生成一个单位。
  - `相近颜色矩形合并`：把相邻且颜色差在容差内的像素合并为一个矩形单位，减少生成实体数。
- 自动计算单个像素块厚度，取 `min(scale.x, scale.z)`。
- 选择白模元件类型，默认使用长方体。

坐标和缩放规则：

| 项 | 规则 |
| --- | --- |
| 图片解析缩放 | 先按用户拖拽或输入的 X/Y 百分比缩放图片，得到解析用图片；后续像素解析基于这张缩放后的图片。 |
| 生成尺寸 | 可以按总高宽反推单块大小，也可以按单块大小反推总高宽；这是米制尺寸，和图片解析缩放是两套参数。 |
| 颜色合并 | 矩形合并模式会从左上到右下扫描，把未处理的相近颜色区域扩展成矩形；矩形颜色取内部像素平均值。 |
| 背景过滤 | 指定背景 RGB 后，只从图片四周出发 BFS 删除与边界连通的相近背景；主体轮廓内部的同色区域会保留。该规则同时作用于逐像素和矩形合并模式。 |
| 坐标中心 | 每个像素单位的 `position` 是该单位包围盒中心，不是底部、左下角或像素角点。 |
| 无缝优先 | 相邻像素单位的中心距严格等于对应方向的 `scale`，优先保证单位间没有缝隙或重叠。 |
| 坐标精度 | `position.x/y/z` 写入前会按 `0.01m` 网格生成，因为元件位置的最小变化量是 `0.01m`。 |
| 尺寸逼近 | 用户输入的是期望高宽；实际总高宽等于 `采样像素数 * 单位尺寸`，会落到 `0.01m` 网格上并尽量接近期望值。 |
| 横向 | 图片宽度映射到 GIA 的 `X` 轴。 |
| 纵向 | 图片高度映射到 GIA 的 `Z` 轴，图片第一行在正 `Z` 方向。 |
| 厚度 | `scale.y` 自动取 `min(scale.x, scale.z)`，不需要手动填写。 |
| 合并矩形厚度 | 矩形合并后只改变 `scale.x/scale.z` 覆盖范围，所有生成单位的 `scale.y` 保持相同。 |
| 缩放 | `scale.x/y/z` 直接代表该单位包围盒三轴实际长度，单位为米，并按 `0.01m` 网格生成。 |

例如 2x2 图片映射到 `2m x 2m` 时，每个像素块的 `scale.x=1`、
`scale.z=1`。左上角像素中心坐标为 `(-0.5, 0, 0.5)`。
