# GIA 双功能参数调节与消融指南

> 适用版本：当前仓库代码快照（2026-07-19）  
> 覆盖功能：**图片生成 GIA**、**线稿生成 GIA**  
> 文档口径：范围、默认值和实际行为均以当前 UI 与处理代码为准。

## 1. 如何阅读本文

本文中的“消融”指**保持输入图片和其他参数不变，只改变一个参数或关闭一个模块，再比较结果变化**。表格中的趋势是依据当前算法机制得出的预期方向，不是假装已经对所有图片跑过的实测数值；不同图片可能因颜色、透明度、噪声和线宽不同而出现局部反例。

建议每次比较至少记录以下指标：

| 指标 | 含义 | 两个功能是否适用 |
|---|---|---|
| GIA 对象数 | 生成的基元总数；通常越少，编辑与加载压力越小 | 是 |
| GIA 文件大小 | 最终 `.gia` 字节数 | 是 |
| 轮廓/颜色保真度 | 与目标图片的视觉接近程度 | 是 |
| 细节保留率 | 小区域、细线、尖角是否仍存在 | 是 |
| 断线/误连接 | 本应连续的线是否断开、相邻线是否被错误连接 | 线稿功能 |
| 覆盖 IoU | 目标线稿区域与矩形带覆盖区域的交并比，越高越好 | 矩形带 |
| 漏覆盖率 | 目标线稿中未被矩形覆盖的比例，越低越好 | 矩形带 |
| 生成耗时 | 从点击生成到结果完成的时间 | 是 |

推荐消融步骤：

1. 固定同一输入图和同一输出尺寸。
2. 先按默认值生成一份基线结果，下载“生成摘要 JSON”。
3. 每轮只调整一个参数；不要同时修改缩放、阈值和预算。
4. 重新生成受影响阶段的预览，再导出 GIA。
5. 对照对象数、文件大小、视觉结果和摘要指标。
6. 找到可接受区间后，再做两个参数的联合微调。

## 2. 图片生成 GIA

### 2.1 当前处理链

```text
上传图片
  → 解析缩放（最近邻）
  → 按像素预算和 0.01 m 网格采样（缩小时使用 Lanczos）
  → Alpha 过滤 / 边缘连通背景移除
  → 逐像素生成 或 相近颜色矩形合并
  → 写入元件类型、碰撞和负载优化参数
  → 最终图片预览 / GIA 导出
```

### 2.2 输入与解析缩放参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 上传图片 | PNG/JPG/JPEG/WebP/BMP | 读取后统一转为 RGBA | 换图不属于严格单变量消融；应固定输入进行对比 |
| 等比缩放图片 | 开 / **默认开** | 锁定 X、Y 为相同倍率 | 关闭后可独立拉伸，形状比例可能改变；对象数取决于缩放后的像素面积 |
| 图片 X 缩放 | 1%～400% / **100%** | 改变进入后续解析的宽度；使用最近邻缩放 | 调高：横向细节和候选像素增加，预处理与采样开销上升；调低：横向细节减少、对象数通常下降 |
| 图片 Y 缩放 | 1%～400% / **100%** | 改变进入后续解析的高度 | 趋势同 X；非等比调整会改变画面比例和像素密度 |

注意：解析缩放只改变进入采样的像素画布，不直接决定最终米制尺寸。若“目标解析像素数”较低，先放大再采样可能仍回到相近的采样尺寸，但滤波结果可能不同。

### 2.3 尺寸控制参数

#### 模式 A：调高宽自动计算单个大小

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 尺寸模式 | **调高宽自动计算单个大小** / 调单个大小自动计算高宽 | 决定由“总尺寸”还是“单像素块尺寸”驱动布局 | 切换模式会改变尺寸求解路径，不建议和采样预算同时消融 |
| 锁定图片宽高比 | 开 / **默认开** | 高度按缩放后图片宽高比自动计算 | 关闭后可独立设宽高；可能产生米制拉伸，但不改变采样图的像素拓扑 |
| 期望总宽度 | 0.01～50.00 m / **10.00 m** | 决定 X 方向总尺寸目标 | 调高：单元宽度增大；采样宽度上限也放宽。调低：可能因 0.01 m 网格限制减少采样列数 |
| 期望总高度 | 0.01～50.00 m / **按宽高比计算** | 决定 Z 方向总尺寸目标 | 趋势同总宽度；关闭比例锁后可独立改变纵向比例 |

#### 模式 B：调单个大小自动计算高宽

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 单个像素块宽度 X | 0.01～50.00 m / **0.10 m** | 采样完成后，用采样列数反推总宽度 | 调高：总宽度增大，最高限制为 50 m；对象数一般不直接变化 |
| 单个像素块高度 Z | 0.01～50.00 m / **0.10 m** | 采样完成后，用采样行数反推总高度 | 调高：总高度增大；对象数一般不直接变化 |

布局以 **0.01 m 网格**为优先约束。相邻基元会优先严丝合缝，因此实际总宽高可能与期望值略有偏差。像素块 Y 向厚度没有单独 UI 参数，当前自动取单元宽度与高度中的较小值。

### 2.4 采样与透明度参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 调低 / 关闭 | 调高 / 开启 |
|---|---|---|---|---|
| 目标解析像素数 | 1～40,000 / **2,000** | 采样图的像素预算上限；同时还受总米制尺寸对应的网格单元数限制 | 对象数、文件大小、耗时下降；细节和小色块更易丢失 | 细节更完整；逐像素模式下对象数和耗时通常显著上升 |
| 透明度过滤阈值 | 1～256 / **1** | 仅保留 `alpha >= 阈值` 的像素 | 保留更多半透明边缘，可能增加对象数和毛边 | 移除更多半透明像素；256 会过滤全部像素并导致无法导出 |
| 最终透明度模式 | 使用原有透明度 / **设置全局统一透明度** | 决定矩形合并时是否把 Alpha 差异纳入合并及输出 | 使用原有：透明渐变保真更好，矩形更难合并 | 统一：忽略 Alpha 差异，有利于合并和降对象数 |
| 全局统一透明度 | 0～255 / **255** | 当前实现仅在“相近颜色矩形合并”中写入统一透明度 | 0～99 会从完全透明逐步增加到近乎不透明 | 当前写入层按百分比处理，100～255 最终都会被限制为 100% 不透明 |

> **当前实现边界：**“全局统一透明度”在逐像素生成模式下不会覆盖原 Alpha；在矩形合并模式下实际按 0～100% 解释，UI 的 101～255 都等价于 100%。做透明度消融时建议使用 0、25、50、75、100，不要使用 128/255 来推断百分比。

### 2.5 像素生成算法与颜色参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 像素生成算法 | 逐像素生成 / **相近颜色矩形合并** | 逐像素：每个可见采样像素生成一个基元；矩形合并：从左上到右下贪心扩展矩形 | 切为逐像素：颜色与透明度最忠实，但对象数通常最高。切为合并：对象数降低，边界更块状 |
| 颜色合并容差 | 0～255 / **8**；仅矩形合并显示 | 逐通道判断当前像素与矩形种子像素的差值；每个 R/G/B 通道都需在容差内。使用原透明度时 Alpha 差也需在容差内 | 调低：颜色边界更准确、矩形更碎、对象数增加 | 调高：更多颜色被平均进同一矩形，对象数下降，渐变和细节更易被抹平 |

矩形合并不是任意形状聚类，而是**按扫描顺序生成轴对齐矩形**；矩形颜色取内部像素平均值。相同容差在噪声图、渐变图和大色块图上的对象数变化会非常不同。

### 2.6 背景移除参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 按 RGB 去掉背景 | 开 / **默认关** | 只从图片四周开始，删除与目标 RGB 接近且四邻域连通的区域 | 关闭：保留完整画布，背景也会生成对象 | 开启：通常显著减少背景对象；主体内部不与边缘连通的同色区域会保留 |
| 背景 R/G/B | 0～255 / **255,255,255** | 指定背景目标色 | 改变任一通道会移动被识别的颜色中心 | 应使用边缘处真实背景色，而非主体内部颜色 |
| 背景容差 | 0～255 / **0** | 每个 RGB 通道分别允许的最大差值 | 调低：删除更保守，可能残留压缩噪点 | 调高：背景删除更完整，但与边缘连通的近色主体也可能被吃掉 |

### 2.7 碰撞、元件和负载优化参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 碰撞模式 | 开启原生碰撞 / 开启碰撞和攀爬 / **关闭碰撞** | 分别写入阻挡、攀爬或都关闭 | 只改变游戏交互字段，不改变几何、颜色和对象数；开启碰撞可能增加运行时物理开销 |
| 生成元件类型 | 长方体、球体、平面、三棱柱、五棱柱、三棱锥、五棱锥、圆柱、圆锥、线框长方体、线框圆柱 / **长方体** | 选择复制的模板元件类型 | 不改变采样对象数，但会改变单个对象的外形、视觉拼接与潜在运行成本 |
| 模板 GIA | 路径文本 / **`UI/白模解析.gia`** | 作为导出的基础模板 | 换模板不是算法消融；模板缺失会直接导出失败 |
| 超出范围仍运行 | 开 / **默认关** | 为每个生成元件写入超出加载范围仍运行标志 | 开启不改变几何和对象数，但远距离对象仍可能执行逻辑，运行负载可能上升 |
| 超出范围显示 | **默认** / 永久显示 / 永久以最高精度显示 | 控制远距离显示策略 | 永久显示扩大可见范围；最高精度显示画质最稳定，但通常也是最重的设置 |
| 输出文件名 | 默认 `<原文件名>_image_pixels.gia` | 只影响下载文件名 | 不影响任何生成结果；省略 `.gia` 时会自动补全 |

### 2.8 图片功能默认消融基线

建议先用以下基线导出，再逐项改变：

```yaml
解析缩放: X=100%, Y=100%, 等比=true
尺寸模式: 调高宽自动计算单个大小
总宽度: 10.00m
锁定宽高比: true
目标解析像素数: 2000
透明度过滤阈值: 1
最终透明度模式: 设置全局统一透明度
全局统一透明度: 255  # 当前等价于 100% 不透明
像素生成算法: 相近颜色矩形合并
颜色合并容差: 8
按RGB去背景: false
碰撞: 关闭
元件: 长方体
超出范围仍运行: false
超出范围显示: 默认
```

优先消融顺序：`目标解析像素数 → 颜色合并容差 → 背景移除 → 透明度阈值 → 输出尺寸`。这组顺序最容易看出“对象数—保真度”的主要拐点。

## 3. 线稿生成 GIA

### 3.1 当前处理链

```text
上传图片
  → 图片缩放
  → 检测边缘 或 上传线稿识别
  → Alpha / 断线连接 / 小连通区域清理
  → 可选：保护曲线、删除曲线、排除短逻辑曲线
  → 可选：宽线细化为单像素中心骨架
  ├─ 曲线拟合 → 简化与基元预算
  └─ 矩形带实验 → OpenCV HoughLinesP + distanceTransform + minAreaRect
  → 导出颜色、叠底、碰撞、负载优化
  → GIA
```

当前实现已经使用现成库完成关键算法：OpenCV Zhang-Suen 细化、`HoughLinesP`、距离变换和最小外接旋转矩形；SciPy 稀疏图连通分量；`simplification` 库曲线简化。

### 3.2 图片缩放参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 等比缩放 | 开 / **默认开** | 锁定 X、Y 缩放一致 | 关闭后可改变线稿比例；米制输出锁比时也会跟随新比例 |
| X 缩放 | 1%～400% / **100%** | 使用 Lanczos 改变处理画布宽度 | 调高：线条拥有更多像素，细节与运算量增加；调低：小间隙、小曲线更易消失 |
| Y 缩放 | 1%～400% / **100%** | 改变处理画布高度 | 趋势同 X；非等比缩放会改变角度、曲线长度和后续 px 阈值的相对尺度 |

所有以 `px` 为单位的阈值都作用于缩放后的画布。因此缩放改变后，应重新校准最小区域、短曲线、短线段、断点距离和最小线宽。

### 3.3 线稿来源与识别模式

| 参数 | 可选范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 线稿来源 | **检测边缘** / 上传线稿图 | 前者从普通图像提取强度边缘；后者按颜色直接识别线稿 | 检测边缘常产生物体轮廓双边；上传线稿更适合已经二值化或颜色明确的素材 |
| 线稿颜色识别 | 指定 RGB / **自动：非 0 值视为线稿** / 直通 | 仅“上传线稿图”显示 | 指定 RGB 最可控；自动模式适合未知黑白极性；直通完全跳过颜色识别和基础像素清理 |

#### 指定 RGB

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 线稿 R/G/B | 0～255 / **0,0,0** | 目标线色 | 设为实际线色可隔离其他颜色；偏离真实线色会漏检 |
| RGB 容差 | 0～255 / **16** | 取 R/G/B 三通道最大差，全部在容差内才算线稿 | 调低：选择更纯净但容易断线；调高：抗压缩噪声更强，但背景近色可能混入 |

#### 自动非 0 模式

| 参数 | 可选范围 / 默认值 | 实际作用 | 单参数消融变化 |
|---|---|---|---|
| 非 0 判定阈值 | 0～254 / **0** | 初始候选为 `max(R,G,B) > 阈值` | 调高：只有更亮像素进入正极性候选；暗灰细线更易丢失 |
| 自动探索黑线/白线极性 | 开 / **默认开** | 比较非零、反相、Otsu 暗区和 Otsu 亮区，按占用率与边缘密度评分选择 | 关闭：速度略快且结果固定，但黑线白底素材很可能把背景当线；开启通常更稳健 |

#### 直通模式

直通会把缩放图原样传给下一阶段，不执行颜色识别、Alpha 过滤、断线连接或小区域过滤。适用于已经准备好的黑线白底图。关闭中心骨架细化时，后续可能把宽线区域当成多条像素路径，基元数明显上升。

### 3.4 检测边缘算子参数

#### 公共参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 调低 | 调高 |
|---|---|---|---|---|
| 边缘检测算子 | Canny、Sobel、Scharr、Laplacian、Prewitt / **Canny** | 选择梯度/二阶响应算法 | 不适用 | 不适用 |
| 检测前高斯模糊核 | 1、3、5、7、9 / **3** | 检测前平滑；1 表示不模糊 | 细节多、噪点和纹理边缘也多 | 噪声少、主轮廓更稳定，但细线和尖锐细节可能消失 |

#### Canny 参数

| 参数 | 范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 低阈值 | 0～255 / **50** | 弱边缘连接阈值 | 降低会保留更多弱边缘和噪声；升高会减少弱连接 |
| 高阈值 | 0～255 / **150** | 强边缘种子阈值 | 降低会增加边缘响应；升高会只保留更明显轮廓，可能断线 |
| Canny 内部 Sobel 核尺寸 | 3、5、7 / **3** | 计算梯度的邻域大小 | 小核定位更精细、对噪声敏感 | 大核响应更平滑、更宽泛，细节定位可能变粗 |

建议保持 `低阈值 < 高阈值`。当前 UI 不强制这一关系，反向设置会产生难以解释的结果。

#### Sobel 参数

| 参数 | 范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 响应阈值 | 0～255 / **45** | 归一化梯度幅值下限 | 降低：边更多、噪声更多；提高：边更少、更强 |
| 卷积核尺寸 | 1、3、5、7 / **3** | 梯度计算邻域 | 增大后更平滑但定位更粗，运行量也略增 |

#### Scharr 参数

| 参数 | 范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 响应阈值 | 0～255 / **55** | Scharr 梯度幅值下限 | 降低增加弱边；提高仅保留强边 |

#### Laplacian 参数

| 参数 | 范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 响应阈值 | 0～255 / **35** | 二阶响应下限 | 降低会出现更多纹理与双边响应；提高会减少弱细节 |
| 卷积核尺寸 | 1、3、5、7 / **3** | 二阶导数邻域 | 增大更强调整体结构，也更可能扩大响应区域 |

#### Prewitt 参数

| 参数 | 范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 响应阈值 | 0～255 / **45** | Prewitt 梯度幅值下限 | 降低增加边缘与噪声；提高减少边缘和对象候选 |

### 3.5 基础清理与短曲线参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 调低 / 关闭 | 调高 / 开启 |
|---|---|---|---|---|
| Alpha 阈值 | 0～255 / **1** | 候选像素需满足 `alpha >= 阈值` | 保留透明边缘 | 排除更多半透明像素，可能造成断线 |
| 断线连接核 | 1、3、5、7 / **1** | 椭圆核形态学闭运算；1 表示不连接 | 保持原始间隙，断线更多 | 填补小缺口，但相邻线可能被误连、孔洞可能被填小 |
| 最小连通区域 | 1～10,000 px / **4 px** | 删除面积小于阈值的 8 邻域连通块 | 保留小点、小字和噪声 | 噪声减少，对象数下降；小装饰和孤立细节会被删除 |
| 排除短曲线 | 开 / **默认开** | 中心化后，在叉点只续接最平滑的一对分支，再按完整逻辑曲线长度删除 | 关闭：保留所有短分支，细节与杂线都更多 | 开启：短毛刺、短树杈减少，结果直接成为删除编辑和下一阶段默认输入 |
| 排除曲线长度 | 0.5～1,000 px / **8.0 px** | 删除长度严格小于阈值的逻辑曲线；保护曲线例外 | 小值保留更多细节和毛刺 | 大值显著简化图形，可能删掉眼睛、文字笔画等短结构 |

“最小连通区域”按**面积**过滤，“排除曲线长度”按中心路径的**长度**过滤，两者不能互相替代。

### 3.6 边缘保护分支与删除编辑

| 参数 / 操作 | 范围 / 默认值 | 实际作用 | 消融变化 |
|---|---|---|---|
| 保护分支数量 | 0～多个 / **0** | 每个分支独立检测、选择要保护的完整逻辑曲线 | 分支增加不直接改变结果，只有确认保护/删除的曲线才生效；分支过多会增加管理和预览计算量 |
| 分支名称 | 文本 | 仅用于辨识分支 | 不影响算法 |
| 分支等比缩放 | 10%～400% / **100%** | 用不同分辨率重新检测分支，随后用归一化坐标映射回主画布 | 调低更粗略；调高更容易选到细曲线，但预览更慢且路径可能更碎 |
| 分支线稿来源 | 边缘检测 / 上传线稿 | 每个保护分支可独立选择来源，与主线稿来源无关 | 可用边缘检测补弱轮廓，也可按 RGB、自动非零或直通方式补入上传线稿 |
| 分支识别与清理参数 | 对应来源的全部参数 | 边缘检测包含算子、阈值、模糊与清理；上传线稿包含 RGB、非零、极性、直通与清理 | 参数更宽松可捕获弱线；更严格可隔离主轮廓 |
| 保护曲线选择 | 点击完整逻辑曲线 | 合并回主结果，并绕过短曲线与拟合前短线段排除 | 保护越多，关键细节越稳，但对象预算压力增大 |
| 分支删除曲线 | 点击完整逻辑曲线 | 从主处理结果删除 | 删除可直接减噪和降对象数；误删会在后续所有阶段持续 |
| 删除原有线条 | 点击主结果逻辑曲线 | 删除当前有效处理结果中的线 | 与分支删除相同；只有点击“确定”才替换后续输入 |

边缘检测和上传线稿共用同一套保护机制、保护分支状态和编辑弹窗。保护分支中的参数范围和默认值与第 3.3～3.5 节一致。切换主线稿来源或点击“生成 / 刷新处理线稿预览”重新生成主线稿时，全部保护分支会清空；主删除选择作为独立编辑记录保留。保护与删除是人工干预变量，做算法消融时应固定选择集合。

保护与删除的输入顺序如下：

```text
主线稿基础识别
  → 开启短曲线排除：使用排除后的结果
    关闭短曲线排除：使用排除前的结果
  → 写入“下一阶段输入”
  → 保护弹窗右侧在该输入上添加线条
  → 删除弹窗左侧读取当前下一阶段输入并删除线条
  → 点击确定后，把编辑结果写回下一阶段输入
```

### 3.7 宽线中心骨架参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 调低 / 关闭 | 调高 / 开启 |
|---|---|---|---|---|
| 自动把宽线细化为单像素中心骨架 | 开 / **默认开** | 使用 OpenCV Zhang-Suen 算法细化 | 关闭：保留原宽度，曲线拟合可能追踪出多条并行路径 | 开启：路径更集中、基元更少，宽线原始厚度丢失 |
| 宽线探索半径 | 0、2、4、6、8 / **4** | 作为最大搜索半径；算法实际比较从 0 到该值的每个整数膨胀候选并自动选最低评分结果 | 小值：候选少、速度快、对破碎宽线修复有限 | 大值：候选多、耗时增加，有机会连通破碎宽线，也可能改变支路结构 |

探索半径不是“强制膨胀半径”。例如选择 4 时会比较 0、1、2、3、4 五个候选，最终仍可能选择 0。

### 3.8 曲线拟合参数

| 参数 | 可选范围 / 默认值 | 实际作用 | 调低 / 关闭 | 调高 / 开启 |
|---|---|---|---|---|
| 曲线合并误差阈值 | 0～100 px / **1.5 px** | `simplification` 曲线简化的初始误差下限；预算超限时算法只会继续提高误差 | 更贴合原曲线、线段更多 | 曲线更直、更简化、对象更少；尖角和小弧线更易失真 |
| 排除线段长度（预算前过滤） | 0～1,000 px / **2.0 px** | 在预算搜索前，按完整骨架路径长度删除短路径；保护路径例外 | 保留短路径和小细节 | 降对象数和毛刺，但可能删除短笔画；不同于前一步“逻辑曲线长度” |
| 期望最大基元数 | 1～200,000 / **500** | 数量目标；允许约 12% 软浮动，必要时提高简化误差 | 小预算迫使更强简化，甚至按长度裁剪 | 大预算允许更多线段，但不会主动把已经较少的线段拆多 |
| 锁定输出宽高比 | 开 / **默认开** | 根据缩放画布比例自动计算 Z 高度 | 关闭后可进行非等比米制拉伸 | 开启保持图形比例 |
| 总宽度 X | 0.01～500 m / **10.00 m** | 像素 X 映射到世界 X 的范围 | 只改变世界尺寸，不改变拟合线段数；会影响 50 m 长度拆分 | 同左 |
| 总高度 Z | 0.01～500 m / **按比例计算** | 像素 Y 映射到世界 Z 的范围 | 只改变世界尺寸与角度比例 | 同左 |
| 线段基元 | 11 种模板 / **长方体** | 选择曲线线段复制的模板类型 | 不改变拟合段数，但改变实际视觉形状 |
| 长边与线边厚度轴 | **X 为长边** / Z 为长边 | 决定模板哪个局部轴沿线段方向 | 几何路径不变；选错会让非对称模板朝向异常 |
| 线边厚度 | 0.0001～50 m / **0.0100 m** | 同时控制平面短轴宽度和 Y 向厚度 | 更细、更轻薄，接头可能显缝 | 更粗、更厚，覆盖增强但相邻线容易重叠 |

软预算上限为 `ceil(期望最大基元数 × 1.12)`。如果初始误差下的线段数已不超过软上限，算法不会为了“凑到目标值”增加线段。世界空间中单段超过 50 m 时会自动拆分，因此最终对象数可能高于拟合线段数。

### 3.9 矩形带拉升（实验）参数

矩形带固定使用“长方体”，从二维线稿区域提取中心线、估计局部宽度，并在导出时沿 Y 轴拉升。它不是像素网格算法。

#### 输入选择规则

- 开启“排除短曲线”时：读取**排除短曲线后的结果图**。
- 关闭“排除短曲线”时：读取**处理线稿预览**，可保留原线宽信息。

这意味着开启短曲线排除后，输入往往已经是单像素骨架，距离变换估计到的线宽会明显变窄；若目标是还原粗细变化，应重点比较“短曲线排除开/关”两组结果。

| 参数 | 可选范围 / 默认值 | 实际作用 | 调低 / 关闭 | 调高 / 开启 |
|---|---|---|---|---|
| 锁定矩形带输出宽高比 | 开 / **默认开** | 保持缩放画布比例 | 关闭可非等比拉伸 | 开启保持原比例 |
| 矩形带总宽度 X | 0.01～500 m / **10.00 m** | 输出世界宽度 | 不改变二维矩形数量，只改变世界尺寸 | 同左 |
| 矩形带总高度 Z | 0.01～500 m / **按比例计算** | 输出世界高度 | 不改变二维矩形数量 | 同左 |
| OpenCV 最大断点连接距离 | 0～64 px / **1.5 px** | 作为 Hough 断线连接距离和候选搜索的初始值；内部取整 | 小值更保守，断裂线可能分成多段或漏检 | 大值更易跨越缺口并减少段数，也可能跨接不相干线条 |
| 最短矩形长度 | 0～1,000 px / **2.0 px** | Hough 最短线段长度，内部最低按 1 px | 小值保留短细节但矩形更多 | 大值减少碎片和噪声，短笔画可能只能依靠补片覆盖 |
| 期望最大矩形基元数 | 1～200,000 / **500** | 目标数量；常规软上限约 112%，覆盖救援最多约 135% | 小值更简化，若覆盖不够仍可能超出软上限 | 大值容纳更多局部矩形，覆盖通常更完整 |
| 识别线宽倍率 | 0.05～10.0 / **1.0** | 将距离变换估计线宽整体乘以倍率 | 矩形变窄、误覆盖少但漏覆盖增加 | 矩形变宽、漏覆盖下降但外溢和重叠增加 |
| 最小识别线宽 | 0.1～500 px / **1.0 px** | 线宽估计的下限 | 细线更细，可能出现缝隙 | 所有细线至少达到该宽度，覆盖增强但细节变胖 |
| 矩形接头重叠 | 0～100 px / **1.0 px** | 每条矩形沿长度方向两端各扩展，合计增加约设定值 | 接头更准确但易出现缝隙 | 缝隙减少，对象间重叠和外溢增加 |
| 最大漏覆盖率 | 0～100% / **1.0%** | 候选优先满足的漏覆盖目标；不足时用旋转补片救援 | 小值要求严格，补片和对象数增加 | 大值允许遗漏细节，对象数可下降 |
| 中心路径探索半径 | 0、2、4、6、8 / **4** | 提取中心路径时的最大候选半径 | 候选少、速度快 | 候选多、可改善破碎宽线，也增加计算量和误连接可能 |
| 长方体长边轴 | **X 为长边** / Z 为长边 | 决定模板局部轴映射 | 不改变拟合覆盖，仅改变模板朝向 |

矩形带会测试最多 14 组逐渐增大的断点容差；“最大断点连接距离”是搜索起点，不保证最终有效值等于输入值。线宽取中心线采样处距离变换宽度的第 80 百分位，再应用倍率和最小线宽。

## 4. 关键联动与推荐消融顺序

### 4.1 图片功能联动

| 联动 | 结果 |
|---|---|
| 解析缩放 × 目标解析像素数 | 预算足够时，放大增加细节；预算很低时，放大结果仍会再次缩小，收益有限 |
| 总米制尺寸 × 目标解析像素数 | 0.01 m 网格限制采样行列数；总尺寸很小时，即使预算很高也不能保留所有像素 |
| 原透明度 × 颜色容差 | Alpha 差异也参与容差判断，半透明渐变会拆出更多矩形 |
| 去背景 × 颜色合并 | 先去背景再合并，通常能显著降低大背景矩形数量 |
| 永久最高精度显示 × 大对象数 | 几何不变，但远距离显示负载风险最大 |

### 4.2 线稿功能联动

| 联动 | 结果 |
|---|---|
| 图片缩放 × 所有 px 阈值 | 缩放越大，同一个物理细节占用的 px 越多；原阈值不能直接照搬 |
| 高斯模糊 × 边缘阈值 | 大模糊会削弱细边，通常要配合更低阈值；否则容易漏边 |
| 断线连接核 × 短曲线排除 | 先连接再按曲线长度清理；大连接核可能把本应删除的短线并入长线而保留下来 |
| 保护曲线 × 两级短线过滤 | 被保护路径绕过“排除短曲线”和“排除线段长度” |
| 中心骨架 × 曲线拟合 | 关闭骨架细化可能把宽线变成多条路径，显著增加拟合基元 |
| 初始误差 × 基元预算 | 初始误差是下限；预算不足时算法会自动提高误差，最终有效值可高于输入值 |
| 排除短曲线 × 矩形带线宽 | 开启后矩形带读取排除后的结果，常已变成单像素骨架，识别线宽会变窄 |
| 漏覆盖目标 × 矩形预算 | 覆盖优先级高于数量；严格漏覆盖目标允许对象数进入约 35% 救援区间 |
| 世界尺寸 × 50 m 单段限制 | 曲线拟合段映射到世界后超过 50 m 会拆分，导出对象数可能高于预览拟合段数 |

推荐顺序：

1. 固定输出米制大小，先确定识别模式和边缘算子。
2. 调阈值与模糊，使主线完整且背景噪声可控。
3. 调断线连接核和最小连通区域。
4. 对比短曲线排除开/关，再选择长度阈值。
5. 确定中心骨架开/关及探索半径。
6. 曲线路线调“误差阈值 → 短线段阈值 → 基元预算”。
7. 矩形带路线调“线宽倍率 → 接头重叠 → 漏覆盖目标 → 基元预算”。
8. 最后只调整世界尺寸、厚度、颜色、碰撞和负载优化。

## 6. 当前实现中需要特别注意的边界

1. **图片全局统一透明度的 UI 范围与写入单位不一致。** 当前只有矩形合并使用该值，并按 0～100% 处理；101～255 都会成为 100%。
2. **逐像素模式保留原始 Alpha。** 即使 UI 选择全局统一透明度，当前逐像素分支仍按原像素 Alpha 导出。
3. **图片实际采样量可能低于“目标解析像素数”。** 它还受图片原尺寸和 0.01 m 米制网格行列上限约束。
4. **线稿基元预算是软目标。** 曲线约允许 12% 浮动；矩形带为满足覆盖可进入约 35% 的救援区间。
5. **宽线探索半径是搜索上限，不是强制值。** 增大它主要增加候选和耗时，最终可能仍选择半径 0。
6. **矩形带断点距离也是搜索起点。** 算法会继续评估更大的候选容差。
7. **排除短曲线会改变矩形带输入。** 开启时使用排除后的结果，关闭时使用处理线稿，二者的线宽信息可能完全不同。
8. **直通模式与短曲线清理存在路径差异。** 直通会跳过阶段 2 清理；若先手动生成了“关闭短曲线清理”的骨架缓存，再进行拟合，结果可能与直接让后续阶段自动补算不同。严格消融时应在改变直通/短曲线设置后依次刷新处理线稿、中心骨架和最终拟合预览。

## 7. 术语说明

| 术语 | 通俗解释 |
|---|---|
| 基元 | GIA 中实际生成的一个元件对象，例如一个长方体像素块或一段线 |
| Alpha | 像素透明度，0 为完全透明，255 为完全不透明 |
| 形态学闭运算 | 先扩张再收缩，用于连接小断口、填补小缝隙 |
| 连通区域 | 在像素邻接关系下连成一块的区域 |
| 中心骨架 | 把宽线压缩成大致位于中心的一像素路径 |
| 曲线简化误差 | 允许简化线段偏离原曲线的最大像素尺度；越大越简化 |
| 软预算 | 目标不是绝对硬上限，允许一定比例浮动以避免严重损坏图形 |
| HoughLinesP | OpenCV 的概率霍夫线段检测，用于从中心路径中找直线段 |
| distanceTransform | 距离变换；这里用中心点到线稿边缘的距离估计局部线宽 |
| minAreaRect | OpenCV 最小面积旋转矩形；这里用于给漏覆盖区域添加补片 |
| IoU | 预测覆盖与目标区域的交集面积除以并集面积，越高越接近 |
| 负载优化 | 当前 UI 中“超出范围仍运行”和“超出范围显示”的组合设置 |
